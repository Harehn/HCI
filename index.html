Project Proposal

We found a certain lack of addressing accessibility concerns from a subset of university websites. We focused on how to address the needs of color blind people in that aspect. We found one of the ways that colorblind people were impeded were the color schemes used in websites. We want to fix it using an option to allow the users to modify the color scheme in the website using a preview.

Design Evolution:

Design Change 1

According to the formative feedback provided, there are some difficulties in navigating the website. The first change we had to implement based on the formative feedback was the discoverability of our buttons. We decided to increase the contrast of the on-screen elements with the background so that they are more visually identifiable and easy to see.

We believe this change will make our prototype better as it will increase the discoverability of the features.

Design Change 2

Another point in the formative feedback was to have more system feedback ie to let the system inform the users about the changes that have just been made. A simple way to do that would be to have a pop-up letting the user know that the settings have been applied. Another way we can increase the user awareness of the applied changes would be to have a preview of the effect of those changes.

We believe this change will be helpful to let the users know about the state and effect of the prototype and help them understand our project more. This is in line with Nelson’s usability goals, #1 visibility of system State.

Design Change 3

The formative feedback by group H also had mentions of the UI elements we had used in our figma prototype were not suitable for the UI we had. The UI elements could be replaced by more appropriate ones. We intend to replace the radio buttons and prevent the users from thinking that we are presenting them with system selection. We believe that a slider or text box might be more appropriate.

We believe this change is useful as this is in line with Nelson’s fifth Heuristic Principle of “Error Prevention”

Design Change 4

In order to make it easier for the users to explore our website, more labels could be added, as suggested by the other group. Moreover, colour usage of the navigation elements could also be considered to help colour blindness locate things faster and easier. We could also make the accessibility control option separated from the main menu, such as locating the right bottom, to make it easy to be identified by the colour blindness.

We believe this change will make our prototype better as it will increase the discoverability of the features.
